# 2026-07-04 21:55:16 Beeper Feedback

## User Goal

Replace the slow screen-flash touch feedback with a short GPIO pulse for an external beeper connected to GPIO23.

## Files Read

- `main/main.c`
- `scripts/build.sh`
- `scripts/flash.sh`
- `scripts/capture-serial.sh`

## Changes Made

- Added `BEEPER_GPIO` on `GPIO_NUM_23`.
- Added `BEEPER_PULSE_MS` set to `80ms`.
- Configured GPIO23 as an output during startup and defaulted it low.
- On each valid touch action, GPIO23 is driven high for 80ms, then low.
- Removed the previous per-touch `lcd_flash_button()` redraw path so the screen no longer flashes on every button press.

## Verification

- Built successfully with `./scripts/build.sh`.
- Flashed successfully to `/dev/cu.usbserial-10` with `./scripts/flash.sh /dev/cu.usbserial-10`.
- Captured serial startup logs with `./scripts/capture-serial.sh /dev/cu.usbserial-10 6`.
- Confirmed the firmware prints:
  - `beeper pin: GPIO23 pulse=80ms`
  - `BLE HID connected`

## Notes

- The current pulse is a simple digital high-level pulse. If the beeper volume or tone is not ideal, the next step is to switch GPIO23 to LEDC/PWM and tune frequency and duty cycle.
