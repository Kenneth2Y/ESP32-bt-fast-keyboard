Timestamp: 2026-07-04 14:47:02 Asia/Shanghai

## User Goal

Build the initial firmware for an ESP32-WROOM-32E touch shortcut keyboard that connects to macOS over Bluetooth HID. The first target is a lightweight Bluetooth keyboard with three touch buttons:

- `C`: macOS copy shortcut, `Command+C`
- `V`: macOS paste shortcut, `Command+V`
- `X`: macOS cut shortcut, `Command+X`

The device must use Bluetooth because the board has no USB OTG keyboard capability. Serial logs should be used for early validation before the display UI is refined.

## Changes Made

- Created a new ESP-IDF project.
- Implemented BLE HID keyboard firmware in `main/main.c`.
- Added resistive touch sampling using the board pins:
  - `RTP_SCK`: GPIO25
  - `RTP_DIN`: GPIO32
  - `RTP_CS`: GPIO33
  - `RTP_IRQ`: GPIO36
  - `RTP_DOUT`: GPIO39
- Mapped the 320x240 touch area into three horizontal button zones: `C`, `V`, `X`.
- Sent macOS shortcuts using HID `Left GUI` modifier, which corresponds to the Mac Command key.
- Added serial logs for boot, Bluetooth advertising/connect/disconnect, touch coordinates, and sent shortcuts.
- Added ESP-IDF helper scripts:

## Key Decisions

- Used BLE HID keyboard instead of USB because the board does not provide USB OTG HID.
- Used `Left GUI + C/V/X` because macOS maps the GUI key to Command.
- Delayed full TFT UI implementation until after Bluetooth and touch behavior are verified through serial logs.
- Removed internal pull-up configuration on GPIO36/GPIO39 because ESP32 input-only pins do not support internal pull-ups.
