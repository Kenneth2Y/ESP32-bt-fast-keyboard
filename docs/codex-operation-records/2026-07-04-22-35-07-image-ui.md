# 2026-07-04 22:35:07 Image UI

## User Goal

Use the two images in `UIresources/` as the display UI:

- connected state image
- disconnected state image

The screen should switch only when Bluetooth connection state changes. Touch presses should not redraw the screen; feedback remains the GPIO23 beeper pulse.

## Files Read

- `UIresources/Fast Keyboard bt on.png`
- `UIresources/Fast Keyboard bt off.png`
- `main/main.c`
- `main/CMakeLists.txt`

## Changes Made

- Downscaled the two 1280x960 PNG resources to 320x240 for the ESP32 display target.
- Converted the 320x240 images to RGB565 C arrays in:
  - `main/ui_assets.c`
  - `main/ui_assets.h`
- Added `ui_assets.c` to `main/CMakeLists.txt`.
- Replaced the hand-drawn UI with full-screen image rendering:
  - `s_ui_bt_off` when Bluetooth HID is not connected
  - `s_ui_bt_on` when Bluetooth HID is connected
- Preserved the existing touch mapping because the current hardware behavior has already been verified:
  - visible left `C`
  - visible middle `V`
  - visible right `X`
  - implemented through the current touch `y`-axis thirds and inversion compensation.
- Kept touch feedback as GPIO23 beeper output only; no image/button refresh happens on touch.
